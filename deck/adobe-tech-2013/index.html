The PhoneGap Story

code, community, and beer

So, let's begin with the story of PhoneGap. Just as a heads up, this is the condensed version and I'll hold back on the stories of our late nights, early mornings, and countless hacking in pubs.


Nitobi

Tiny web consultancy

Vancouver, Canada

Founded in 1998



The story of PhoneGap begins in 2008 in Vancouver, Canada. Here, in a sketchy part of the downtown district, you'll find a tiny web consultancy called Nitobi. For about a decade, Nitobi had been focused on the desktop web, but in 2008 we decided to get into the mobile market.

Mobile apps

Got into it around 2008

Objective-C for iOS

Java Dalvik for Android

Java ME for BlackBerry

.NET for Windows CE

HTML for webOS

To kick things off, we started learning how to create mobile applications. We learned Objective-C for iOS, Java Dalvik for Android, a different Java for BlackBerry, .NET for Windows CE, and thankfully HTML for webOS. It was a huge time investment and we noticed one frustrating fact: clients wanted the same application on every device.

People want the same app
on every device.

"People wanted the same application on every device." This means developers are burning their time writing the same code in different languages. Personally, I like to write code, but I don't like to solve the same problem twice.



Web

Solved this problem long ago

Websites look the same everywhere




Coming from a web background, we thought the native SDK landscape was absurd. The web has solved this problem a long time ago. A website be accessed by any device with a browser. And... all smartphones have browsers. So, we have to ask ourselves: "Why are we not using the web for app development?"

How it works

Native application

Fullscreen browser

Without an interface

Loading a local HTML file

Binding JavaScript-to-Native



From this very problem, PhoneGap was born. As a concept, PhoneGap is extremely simple. It's a native application, such as iOS, Android, or even Windows or Mac; And it's simply an instance of a fullscreen browser; Without the chrome; That loads an HTML file bundled with the application. The result is an installable application built with web technology. It's that simple.

Fast forward to 2011

500,000 developers

Now let's fast-forward to 2011, skipping all of the late nights, beers, and some incredible hacks. We now have over half a million developers building applications and we support seven different platforms. This was done with a team of 20 people, most of whom were working on client work during the day and PhoneGap in the evening.

Apache Foundation

Donated PhoneGap

Branded Apache Cordova

Gained even more contributors

Near the end of 2011, a lot of that hard work paid off. We had the honour of donating PhoneGap to the Apache Foundation as Apache Cordova. We actually had Sam Ruby, the Director of Apache, put his name down as a sponsor for our project. By moving to Apache, we've opened a floodgate of contribution opportunities and now have companies such as Adobe, IBM, BlackBerry, Google, Microsoft, Mozilla, Sony, and Intel working full-time on our project. It's an absolute honour to work along side the people who built GMail, PHP, and the Apache Web Server.


Adobe

Acquired Nitobi

A shot of steroids to the project



And around the same time, we at Nitobi joined Adobe. This has essentially put the project on steroids because our team can now work full-time on open source. We also have access to Adobe's massive audience, knowledgeable evangelists, and awesome Web Platform team.

PhoneGap Today

in three parts

In a nutshell, that's how PhoneGap came to be. Today, PhoneGap is now composed of three components.

Part one

Apache Cordova

Framework to create apps with
HTML, CSS, and JavaScript

First off is Apache Cordova, an open source project for creating application using HTML, CSS, and JavaScript. This is the original PhoneGap source code and best thought of as an engine analogous to WebKit.

Part two

Adobe PhoneGap

Distribution of Apache Cordova

with added sugar

Second is PhoneGap, an open source distribution of Apache Cordova with some added sugar. In many ways, you can think of this as Google Chrome with respect to WebKit.

Part three

Adobe PhoneGap/Build

Cloud service for compiling PhoneGap apps

And third is PhoneGap/Build, an Adobe cloud service for compiling PhoneGap applications. A huge pain with PhoneGap is that you still need to compile the native code; PhoneGap/Build does this for you.
